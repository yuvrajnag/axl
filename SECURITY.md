# Security Policy

## Supported Versions

Currently, only the latest `main` branch version of AXL is supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| v0.2.x  | :white_check_mark: |
| < 0.2.0 | :x:                |

## Reporting a Vulnerability

We take the security of AXL and our users' applications very seriously. 

If you discover a security vulnerability within AXL (such as a bypass in the permission engine, an issue with OTP confirmation gates, or rate-limiting evasion), please do not disclose it publicly.

Instead, please send an email to `security@axl.dev` (placeholder) or open a confidential Security Advisory on GitHub.

We will acknowledge receipt of your vulnerability report, and we strive to send you regular updates about our progress. If you report a vulnerability, we will coordinate with you to publish an advisory once a fix is available.
